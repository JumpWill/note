# Linux 内核参数优化

## 一、内核参数概述

Linux 内核参数存放在 `/proc/sys/` 目录下,使用 `sysctl` 命令进行查看和修改。大部分参数可以通过以下两种方式持久化:

- **临时修改**(重启失效):

```bash
sysctl -w net.core.somaxconn=65535
```

- **永久修改**:写入 `/etc/sysctl.conf` 或 `/etc/sysctl.d/*.conf`,然后执行 `sysctl -p` 生效。

```bash
echo "net.core.somaxconn = 65535" >> /etc/sysctl.conf
sysctl -p
```

### 查看参数

```bash
# 查看单个参数
sysctl net.core.somaxconn

# 查看所有参数
sysctl -a

# 按前缀过滤
sysctl -a | grep net.ipv4.tcp

# 查看当前文件/proc/sys/ 下的值
cat /proc/sys/net/core/somaxconn
```

## 二、网络相关参数(最常用)

### 1. TCP 连接队列

#### `net.core.somaxconn`

- **含义**:socket `listen()` 函数的 backlog 上限。即已完成三次握手但还未被应用层 `accept()` 的连接队列最大长度。
- **默认值**:128(或 4096,因版本而异)
- **影响**:Web 服务器(nginx/apache)在突发流量下,若 `somaxconn` 太小,新连接会被丢弃,客户端表现为 connection refused/timeout。
- **优化建议**:高并发 Web 服务设为 `32768` 或更高。

```bash
net.core.somaxconn = 32768
```

> ⚠️ 同时需要修改 nginx 的 `listen 80 backlog=32768;` 参数,内核层才会真正接受这么大的 backlog。

#### `net.ipv4.tcp_max_syn_backlog`

- **含义**:SYN 队列(半连接队列)的长度,即收到 SYN 但还没完成三次握手的连接。
- **默认值**:128 或 256
- **优化建议**:遭受 SYN Flood 攻击或高并发短连接场景需调大。

```bash
net.ipv4.tcp_max_syn_backlog = 65535
```

---

### 2. TIME_WAIT 处理

#### `net.ipv4.tcp_tw_reuse`

- **含义**:是否允许将 TIME_WAIT 状态的 socket 重新用于新的对外连接(只能用于客户端主动发起连接的场景)。
- **默认值**:0(关闭)
- **注意**:`tcp_tw_reuse = 1` 是相对安全的,**不要** 启用已经废弃的 `tcp_tw_recycle`(在 NAT 环境下会有问题)。

```bash
net.ipv4.tcp_tw_reuse = 1
```

#### `net.ipv4.tcp_fin_timeout`

- **含义**:套接字由 FIN_WAIT2 状态进入 TIME_WAIT 状态的等待时间(秒)。
- **默认值**:60
- **优化建议**:可降低到 `15`~`30`,减少 TIME_WAIT 数量。

```bash
net.ipv4.tcp_fin_timeout = 15
```

#### `net.ipv4.tcp_max_tw_buckets`

- **含义**:系统同时保持的 TIME_WAIT 套接字最大数量,超过该数会直接销毁并打印警告。
- **默认值**:4096 或 8192
- **优化建议**:高并发短连接服务可调大到 `200000`,避免告警。

```bash
net.ipv4.tcp_max_tw_buckets = 200000
```

---

### 3. Keepalive(长连接保活)

#### `net.ipv4.tcp_keepalive_time`

- **含义**:连接空闲多久后开始发送 keepalive 探测包(秒)。
- **默认值**:7200(2 小时)

#### `net.ipv4.tcp_keepalive_intvl`

- **含义**:探测包发送间隔(秒)。
- **默认值**:75

#### `net.ipv4.tcp_keepalive_probes`

- **含义**:探测失败多少次后认为连接已死。
- **默认值**:9

**优化示例**:Web 服务希望 5 分钟无活动就检测死连接:

```bash
net.ipv4.tcp_keepalive_time = 300
net.ipv4.tcp_keepalive_intvl = 30
net.ipv4.tcp_keepalive_probes = 3
```

> 大多数应用(nginx、tomcat、MySQL)有自己的 keepalive 参数,内核参数是兜底。

---

### 4. Socket 缓冲区

#### `net.core.rmem_max` / `net.core.wmem_max`

- **含义**:所有协议 socket 接收/发送缓冲区的最大字节数。
- **默认值**:`rmem_max=212992`, `wmem_max=212992`

#### `net.ipv4.tcp_rmem` / `net.ipv4.tcp_wmem`

- **含义**:TCP 套接字缓冲区自动调优的三个值(min default max,单位字节)。
- **默认值**:`tcp_rmem = 4096 87380 6291456`, `tcp_wmem = 4096 16384 4194304`

#### `net.core.rmem_default` / `net.core.wmem_default`

- **含义**:socket 创建时的默认缓冲区大小。

#### `net.ipv4.udp_rmem_min` / `net.ipv4.udp_wmem_min`

- **含义**:UDP 套接字的最小缓冲区。

**优化示例**:高带宽长肥管道(LFN,带宽时延积大):

```bash
# 最大接收/发送缓冲区
net.core.rmem_max = 16777216
net.core.wmem_max = 16777216

# TCP 自动调优
net.ipv4.tcp_rmem = 4096 87380 16777216
net.ipv4.tcp_wmem = 4096 65536 16777216

# 默认值
net.core.rmem_default = 262144
net.core.wmem_default = 262144
```

> 缓冲区调大可显著提升大文件传输、千兆网卡下的吞吐。但占用内存也会增加,需要权衡。

---

### 5. 端口范围

#### `net.ipv4.ip_local_port_range`

- **含义**:对外发起连接时可用的本地端口范围。
- **默认值**:`32768 60999`(约 28K 端口)
- **优化建议**:高并发短连接的客户端(如 nginx 反代、爬虫)需要更大的端口范围。

```bash
net.ipv4.ip_local_port_range = 1024 65535
```

---

### 6. 拥塞控制

#### `net.ipv4.tcp_congestion_control`

- **含义**:TCP 拥塞控制算法。可选 `cubic`(默认)、`bbr`、`reno`、`bic` 等。
- **BBR** 由 Google 提出,在高带宽、高丢包率环境下表现优于 CUBIC。

```bash
# 加载 BBR 模块
modprobe tcp_bbr
echo "tcp_bbr" >> /etc/modules-load.d/modules.conf

# 启用 BBR
net.ipv4.tcp_congestion_control = bbr
net.core.default_qdisc = fq
```

> ⚠️ BBR 在内核 4.9+ 才可用,CentOS 7 默认内核不支持,需要升级。

#### `net.ipv4.tcp_slow_start_after_idle`

- **含义**:连接空闲一段时间后是否重新进入慢启动。
- **默认值**:1(开启)
- **建议**:长连接复用的服务(如 RPC)可关闭,避免每次发送突发流量都重新探测带宽。

```bash
net.ipv4.tcp_slow_start_after_idle = 0
```

---

### 7. 其他网络优化

#### `net.ipv4.tcp_fastopen`

- **含义**:TCP Fast Open,在 SYN 包中携带数据,减少 1 个 RTT。
- **取值**:0 关闭,1 客户端启用,2 服务端启用,3 全部启用。

```bash
net.ipv4.tcp_fastopen = 3
```

#### `net.ipv4.tcp_syncookies`

- **含义**:当 SYN 队列满时,启用 SYN Cookies 防止 SYN Flood 攻击。
- **取值**:0 关闭,1 当队列溢出时启用,2 始终启用。

```bash
net.ipv4.tcp_syncookies = 1
```

#### `net.ipv4.tcp_no_metrics_save`

- **含义**:关闭 TCP 连接时是否保存连接指标(如 srtt、rttvar)。
- **建议**:对短连接关闭,避免每次新建连接都重传这些指标,提升性能。

```bash
net.ipv4.tcp_no_metrics_save = 1
```

#### `net.ipv4.tcp_mtu_probing`

- **含义**:是否启用 MTU 探测。
- **取值**:0 关闭,1 被动探测(出现 ICMP need frag 时启用),2 始终主动探测。
- **建议**:在 VPN/PPPoE 等 MTU 异常环境下设为 `1`。

```bash
net.ipv4.tcp_mtu_probing = 1
```

#### `net.ipv4.conf.all.rp_filter`

- **含义**:反向路径过滤,验证数据包源 IP 是否经过接收它的网卡。
- **取值**:0 关闭,1 严格模式,2 宽松模式。
- **建议**:单网卡服务器设为 `1`;多网卡/非对称路由设为 `2`。

```bash
net.ipv4.conf.all.rp_filter = 1
net.ipv4.conf.default.rp_filter = 1
```

## 三、文件系统相关

### 1. 文件句柄

#### `fs.file-max`

- **含义**:系统级最大文件句柄数(包括 socket、文件、管道等)。
- **默认值**:内核根据内存自动计算,通常是 `8192 * 内存页数 / 256`。
- **查看**: `cat /proc/sys/fs/file-nr`(已分配/空闲/最大)

```bash
# 高并发服务需要调大
fs.file-max = 2097152
```

#### `fs.nr_open`

- **含义**:单个进程可打开的最大文件描述符数。
- **默认值**:1048576
- **注意**:`ulimit -n` 的硬限制由 `fs.nr_open` 决定,但 `ulimit -n` 通常更小。

#### `fs.file-nr` 内核自动维护

`/proc/sys/fs/file-nr` 三个数字:已分配句柄数、空闲句柄数、最大句柄数。**勿手动修改**。

### 2. 异步 IO

#### `fs.aio-max-nr`

- **含义**:系统最大并发异步 IO 请求数。
- **默认值**:65536
- **影响**:MySQL InnoDB 等使用 libaio 的数据库需要调大。

```bash
fs.aio-max-nr = 1048576
```

### 3. inotify

#### `fs.inotify.max_user_watches`

- **含义**:每个用户可监控的文件/目录总数。
- **默认值**:8192

#### `fs.inotify.max_user_instances`

- **含义**:每个用户可创建的 inotify 实例数。
- **默认值**:128

**示例**:开发工具(IDE、Nodemon、Docker Desktop)会消耗大量 inotify 资源。

```bash
fs.inotify.max_user_watches = 524288
fs.inotify.max_user_instances = 512
```

## 四、内存相关

### 1. Swap 行为

#### `vm.swappiness`

- **含义**:内核将内存页换出到 swap 的倾向(0~100)。值越大,越积极使用 swap;值越小,越倾向于回收 page cache。
- **默认值**:60
- **优化建议**:
  - **数据库服务器**(MySQL、PostgreSQL):设为 `1` 或 `10`,尽量避免 swap。
  - **桌面**:保持默认 60。
  - **完全禁用 swap**:设为 `0`(不推荐,OOM 时可能丢失数据)。

```bash
vm.swappiness = 10
```

### 2. 脏页回写

#### `vm.dirty_ratio`

- **含义**:当脏页占内存比例达到该值时,**同步**写回(阻塞进程写)。单位:百分比。
- **默认值**:20

#### `vm.dirty_background_ratio`

- **含义**:当脏页占内存比例达到该值时,**后台**开始异步回写。单位:百分比。
- **默认值**:10

#### `vm.dirty_expire_centisecs`

- **含义**:脏页存活多久后开始回写(单位:百分之一秒)。
- **默认值**:3000(30 秒)

#### `vm.dirty_writeback_centisecs`

- **含义**:后台写回线程唤醒间隔(单位:百分之一秒)。
- **默认值**:500(5 秒)

**优化示例**:高写入负载(如数据库、kafka)希望更积极的脏页回写,避免突发 IO:

```bash
# 数据中心 SSD,可激进刷盘
vm.dirty_ratio = 40
vm.dirty_background_ratio = 10
vm.dirty_expire_centisecs = 3000
vm.dirty_writeback_centisecs = 100
```

### 3. 内存分配策略

#### `vm.overcommit_memory`

- **含义**:内核是否允许超额分配内存。
- **取值**:
  - `0`:启发式(默认,拒绝明显错误的申请)
  - `1`:总是允许
  - `2`:不允许超过 `CommitLimit`
- **典型场景**:Redis 启动时提示 `WARNING overcommit_memory is set to 0`,建议设为 `1`。

```bash
vm.overcommit_memory = 1
```

#### `vm.overcommit_ratio`

- **含义**:`overcommit_memory=2` 时,允许申请的内存比例。
- **默认值**:50

### 4. 最小空闲内存

#### `vm.min_free_kbytes`

- **含义**:内核强制保留的最小空闲内存(KB),用于满足突发分配需求。
- **默认值**:内存大小动态计算,通常很小。
- **影响**:设置过大可能导致系统总内存利用率低;过小则容易触发直接回收造成卡顿。

```bash
# 大内存服务器(> 64G)建议设大一点,避免直接回收
vm.min_free_kbytes = 1048576  # 1G
```

### 5. NUMA 相关

#### `vm.zone_reclaim_mode`

- **含义**:NUMA 节点内存回收策略。
- **取值**:
  - `0`:关闭 zone reclaim(默认)
  - `1`:启用 zone reclaim
  - `2`:激进回收
  - `4`:回收页本地回写
- **建议**:MySQL 等内存数据库建议保持 `0`,避免跨 NUMA 访问延迟。

```bash
vm.zone_reclaim_mode = 0
```

## 五、进程 / 信号 / IPC

### 1. 进程与线程

#### `kernel.pid_max`

- **含义**:系统最大 PID 值。
- **默认值**:32768(可调到 4194304)

#### `kernel.threads-max`

- **含义**:系统最大线程数。
- **默认值**:内存动态计算

```bash
# 高并发容器/进程
kernel.pid_max = 4194304
kernel.threads-max = 1000000
```

### 2. 共享内存

#### `kernel.shmmax`

- **含义**:单个共享内存段的最大字节数。
- **默认值**:32 GB(部分老内核是 32 MB)
- **影响**:PostgreSQL、Oracle 等需要大 SGA 的数据库需调整。

#### `kernel.shmall`

- **含义**:系统总共可使用的共享内存页数。
- **单位**:页(通常 4 KB)

```bash
kernel.shmmax = 68719476736   # 64GB
kernel.shmall = 4294967296    # 16 TB
```

### 3. 信号量与消息队列

```bash
# 信号量: SEMMSL SEMMNS SEMOPM SEMMNI
kernel.sem = 250 32000 100 128

# 消息队列
kernel.msgmax = 65536
kernel.msgmnb = 65536
kernel.msgmni = 256
```

### 4. 核心转储

#### `kernel.core_pattern`

- **含义**:core dump 文件命名规则。
- **示例**: `/tmp/cores/core.%e.%p.%t`(程序名、PID、时间戳)

```bash
kernel.core_pattern = /var/core/core.%e.%p.%t
```

#### `kernel.core_uses_pid`

- **含义**:core 文件名是否包含 PID。

#### `fs.suid_dumpable`

- **含义**:SUID 程序是否允许 core dump。
- **取值**:
  - `0`:禁止
  - `1`:允许但丢弃 root 权限
  - `2`:完全允许

## 六、安全相关

### 1. 内核地址随机化

#### `kernel.randomize_va_space`

- **含义**:地址空间布局随机化(ASLR)是否启用。
- **取值**:
  - `0`:关闭
  - `1`:半随机(仅 stack)
  - `2`:完全随机(stack、libc、mmap、heap)

```bash
# 关闭会降低安全性,仅在调试时关闭
kernel.randomize_va_space = 2
```

### 2. 内核信息泄露

#### `kernel.kptr_restrict`

- **含义**:是否限制非特权用户读取 `/proc/kallsyms` 中的内核地址。
- **取值**:
  - `0`:所有人可读
  - `1`:仅 root 可读
  - `2`:完全不可读

#### `kernel.dmesg_restrict`

- **含义**:非特权用户是否可以读取 dmesg 日志(可能泄露硬件/驱动信息)。

```bash
kernel.kptr_restrict = 2
kernel.dmesg_restrict = 1
```

### 3. ptrace 限制

#### `kernel.yama.ptrace_scope`

- **含义**:限制进程 ptrace 调试的范围。
- **取值**:
  - `0`:经典模式
  - `1`:只能调试自己的子进程
  - `2`:仅 root 可调试,普通用户受限
  - `3`:禁止任何 ptrace

```bash
kernel.yama.ptrace_scope = 1
```

### 4. ICMP 与网络防护

#### `net.ipv4.icmp_echo_ignore_all`

- **含义**:是否忽略所有 ICMP echo(ping)请求。设为 `1` 后服务器无法被 ping 通。

#### `net.ipv4.icmp_echo_ignore_broadcasts`

- **含义**:是否忽略广播 ICMP echo(防 smurf 攻击)。

#### `net.ipv4.conf.all.accept_redirects`

- **含义**:是否接受 ICMP 重定向。
- **取值**:
  - `0`:不接受(推荐)
  - `1`:接受

#### `net.ipv4.conf.all.send_redirects`

- **含义**:是否发送 ICMP 重定向。本机作为网关时才需要。

#### `net.ipv4.conf.all.accept_source_route`

- **含义**:是否接受源路由包(几乎用不到,建议关闭,防止 IP 欺骗)。

#### `net.ipv4.conf.all.secure_redirects`

- **含义**:是否接受来自网关列表中网关的重定向。

#### `net.ipv4.conf.all.log_martians`

- **含义**:是否记录不合法的包到日志。

**服务器安全加固模板**:

```bash
net.ipv4.icmp_echo_ignore_broadcasts = 1
net.ipv4.conf.all.accept_redirects = 0
net.ipv4.conf.default.accept_redirects = 0
net.ipv4.conf.all.secure_redirects = 0
net.ipv4.conf.default.secure_redirects = 0
net.ipv4.conf.all.send_redirects = 0
net.ipv4.conf.default.send_redirects = 0
net.ipv4.conf.all.accept_source_route = 0
net.ipv4.conf.default.accept_source_route = 0
net.ipv4.conf.all.log_martians = 1
```

### 5. IPv6 关闭(可选)

如果服务器完全不需要 IPv6,可关闭:

```bash
net.ipv6.conf.all.disable_ipv6 = 1
net.ipv6.conf.default.disable_ipv6 = 1
```

## 七、典型场景优化模板

### 场景 1:高并发 Nginx Web 服务器

```bash
# /etc/sysctl.d/99-nginx.conf

# TCP 连接队列
net.core.somaxconn = 32768
net.core.netdev_max_backlog = 32768
net.ipv4.tcp_max_syn_backlog = 65535

# TIME_WAIT 处理
net.ipv4.tcp_tw_reuse = 1
net.ipv4.tcp_fin_timeout = 15
net.ipv4.tcp_max_tw_buckets = 200000

# 端口范围
net.ipv4.ip_local_port_range = 1024 65535

# Socket 缓冲
net.core.rmem_max = 16777216
net.core.wmem_max = 16777216
net.ipv4.tcp_rmem = 4096 87380 16777216
net.ipv4.tcp_wmem = 4096 65536 16777216

# 文件句柄
fs.file-max = 2097152
fs.nr_open = 2097152

# 连接优化
net.ipv4.tcp_fastopen = 3
net.ipv4.tcp_no_metrics_save = 1
net.ipv4.tcp_slow_start_after_idle = 0

# 安全
net.ipv4.tcp_syncookies = 1
net.ipv4.conf.all.rp_filter = 1
```

应用:`ulimit -n 1048576` 也要同步设置。

---

### 场景 2:MySQL 数据库服务器

```bash
# /etc/sysctl.d/99-mysql.conf

# 内存:尽量避免 swap
vm.swappiness = 1
vm.dirty_ratio = 40
vm.dirty_background_ratio = 10

# AIO
fs.aio-max-nr = 1048576

# NUMA:关闭 zone reclaim
vm.zone_reclaim_mode = 0

# 大页(共享内存)
kernel.shmmax = 68719476736

# 文件句柄
fs.file-max = 2097152

# 网络
net.core.somaxconn = 32768
net.ipv4.tcp_tw_reuse = 1
net.ipv4.tcp_fin_timeout = 15

# 内存分配:Redis 等 fork 型应用
vm.overcommit_memory = 1
```

---

### 场景 3:Redis 服务器

```bash
# /etc/sysctl.d/99-redis.conf

# 关键:overcommit
vm.overcommit_memory = 1

# 关闭 THP(透明大页)— 通过内核启动参数设置:
# transparent_hugepage=never

# 避免 swap
vm.swappiness = 1

# 文件句柄
fs.file-max = 2097152
net.core.somaxconn = 32768
```

## 八、常用调优命令与技巧

### 1. 让修改立即生效

```bash
# 重新加载所有配置文件
sysctl --system

# 重新加载指定文件
sysctl -p /etc/sysctl.d/99-nginx.conf
```

### 2. 修改 ulimit

`/etc/security/limits.conf`:

```text
*    soft    nofile    65535
*    hard    nofile    1048576
root soft    nofile    65535
root hard    nofile    1048576
```

### 3. 修改 PAM 限制

`/etc/pam.d/common-session` / `/etc/pam.d/common-session-noninteractive` 添加:

```text
session required pam_limits.so
```

### 4. 验证参数

```bash
# 当前生效值
sysctl net.core.somaxconn

# /proc/sys/ 下的真实文件
cat /proc/sys/net/core/somaxconn

# 查看生效的配置文件
sysctl -p 2>&1 | grep "net.core"
```

### 5. 内核启动参数

通过 `grubby`(RHEL) 或 `update-grub`(Debian) 设置:

```bash
# 关闭透明大页
grubby --update-kernel=ALL --args="transparent_hugepage=never"

# 启用大页预留
grubby --update-kernel=ALL --args="hugepagesz=2M hugepages=1024"

# 重启生效
reboot
```

查看当前启动参数:

```bash
cat /proc/cmdline
```

## 九、调试与监控

### 1. 查看内核参数变更日志

```bash
# RHEL/CentOS
rpm -q --changelog kernel | less
```

### 2. 监控关键指标

```bash
# 查看 TIME_WAIT 数量
ss -tan | awk '{print $1}' | sort | uniq -c

# 查看文件句柄使用
cat /proc/sys/fs/file-nr
lsof | wc -l

# 查看 TCP 状态分布
ss -s

# 查看 socket 缓冲区使用
ss -tm  # 显示 TCP 内存使用
```

### 3. 性能问题排查清单

| 现象 | 检查参数 | 命令 |
| --- | --- | --- |
| 连接被拒绝 | `somaxconn` | `netstat -s \| grep LISTEN` |
| 大量 TIME_WAIT | `tcp_tw_*` | `ss -tan state time-wait \| wc -l` |
| 大量 CLOSE_WAIT | 应用未关连接 | 应用代码 |
| 内存耗尽 | `vm.swappiness` | `free -h` |
| 高 IO 延迟 | `dirty_*` | `iostat -x 1` |
| OOM kill | `overcommit_memory` | `dmesg \| grep -i oom` |
