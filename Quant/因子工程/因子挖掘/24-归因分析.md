# 第26章 因子绩效归因：Brinson归因、Barra归因模型、因子暴露分解、收益归因报告生成

因子挖出来，策略跑起来，然后呢？

很多人觉得收益好就完事了。但我告诉你，**不知道钱怎么赚来的，迟早会不知道怎么亏回去**。绩效归因，就是帮你搞清楚：这波收益到底是运气，还是实力？

### 26.1 为什么需要绩效归因？

说白了，归因就是拆解收益的来源。我见过太多人，回测曲线漂亮得不行，实盘一塌糊涂。为什么？因为回测里的收益可能来自某几个因子的偶然暴露，而不是策略本身的选股能力。

归因能回答三个核心问题：

- **收益从哪来？**——行业配置？个股选择？还是因子暴露？
- **风险在哪？**——哪些因子在拖后腿？
- **基金经理/策略有没有超额能力？**——剥离了基准之后，还剩多少alpha？

> **核心观点：**归因不是事后诸葛亮，而是事前风控和事后复盘的标准动作。不做归因的策略，就像闭着眼睛开车。

### 26.2 Brinson归因：最经典的业绩分解法

Brinson归因最早用于基金业绩分析，核心思想是把超额收益拆成两部分：**资产配置收益**和**证券选择收益**。

公式很简单：

```text
超额收益 = 配置收益 + 选择收益 + 交互收益

配置收益 = Σ(组合权重_i - 基准权重_i) × 基准收益_i
选择收益 = Σ(组合收益_i - 基准收益_i) × 基准权重_i
交互收益 = Σ(组合权重_i - 基准权重_i) × (组合收益_i - 基准收益_i)
```

嗯，这里要注意：交互收益通常很小，很多分析会把它合并到选择收益里。我个人习惯直接看配置+选择，简单明了。

**我在项目中遇到过**一个案例：某策略超额收益很高，Brinson一拆，发现80%来自配置收益——说白了就是重仓了某个大涨的行业。这跟选股能力没啥关系，换个人来重仓也能赚。这种策略，你敢信吗？

### 26.3 Barra归因：因子层面的精细拆解

Brinson只能拆到行业层面，但因子投资需要更细的粒度。Barra归因模型就是干这个的。

它的核心逻辑是：**组合收益 = 因子暴露 × 因子收益 + 特异收益**

具体步骤：

1. **因子暴露计算**：计算组合在每个Barra因子上的暴露度（比如市值、动量、波动率等）
2. **因子收益估计**：用截面回归估计每个因子的当期收益
3. **收益分解**：把组合收益拆成因子贡献 + 特异收益（残差）

```python
# 伪代码示例：Barra归因核心逻辑
import numpy as np
import pandas as pd

def barra_attribution(portfolio_exposures, factor_returns):
    """
    portfolio_exposures: DataFrame, 组合在因子上的暴露 (n_assets × n_factors)
    factor_returns: Series, 因子收益 (n_factors,)
    """
    # 因子贡献收益
    factor_contribution = portfolio_exposures.dot(factor_returns)

    # 特异收益 = 组合实际收益 - 因子贡献收益
    specific_return = portfolio_returns - factor_contribution

    return factor_contribution, specific_return
```

> **实战技巧：**Barra归因里有个坑——因子收益的估计方法。我建议用市值加权回归，而不是等权。因为大市值股票对市场影响更大，等权会高估小盘因子的贡献。

### 26.4 因子暴露分解：看看你的组合到底暴露了什么

因子暴露分解，说白了就是回答一个问题：**我的组合在哪些因子上有敞口？**

举个例子，你做了一个多因子选股策略，回测收益很好。但Barra一拆，发现组合在市值因子上有巨大的负暴露——也就是说，你其实是在做小盘股策略。如果小盘风格反转，你的收益就会回吐。

暴露分解的步骤：

- **标准化暴露**：把原始因子值转为z-score，方便跨因子比较
- **组合加权暴露**：按持仓权重计算组合的整体暴露
- **暴露归因**：看每个因子对组合风险的贡献占比

```python
# 因子暴露分解示例
def factor_exposure_decomposition(portfolio_weights, factor_matrix):
    """
    portfolio_weights: 组合权重 (n_assets,)
    factor_matrix: 因子暴露矩阵 (n_assets × n_factors)
    """
    # 组合因子暴露 = 权重 × 因子暴露
    portfolio_exposure = portfolio_weights.T @ factor_matrix

    # 风险贡献分解（假设因子协方差已知）
    factor_cov = np.cov(factor_matrix.T)
    risk_contribution = (portfolio_exposure @ factor_cov) * portfolio_exposure

    return portfolio_exposure, risk_contribution
```

> **我曾经踩过的坑：**因子暴露分解时，一定要做**正交化处理**。因子之间往往高度相关（比如价值和低波），不做正交化，暴露度会被重复计算。我早期做归因时，发现市值和动量的暴露加起来超过100%，后来才意识到是共线性问题。

### 26.5 收益归因报告生成：把分析结果可视化

归因分析做完了，怎么呈现？我建议至少包含以下内容：

- **收益拆解饼图**：配置收益、选择收益、因子贡献、特异收益的占比
- **因子暴露热力图**：展示组合在各因子上的暴露度（颜色深浅表示大小）
- **累计归因曲线**：时间序列上，因子贡献收益 vs 组合实际收益的对比
- **风险贡献分解**：每个因子对组合总风险的贡献百分比

下面我用图画一张归因报告的核心结构图，帮你理清逻辑：

```mermaid
flowchart TD
    A[组合收益 + 基准收益] --> B[Brinson归因<br/>配置 + 选择 + 交互]
    A --> C[Barra归因<br/>因子暴露 × 因子收益]
    A --> D[暴露分解<br/>风险贡献 + 暴露度]
    B --> E[归因报告输出<br/>收益拆解饼图 | 因子暴露热力图 | 累计归因曲线 | 风险贡献分解<br/>特异收益分析 | 行业配置贡献 | 风格因子暴露]
    C --> E
    D --> E
    E --> F[策略优化决策：调权重 / 换因子 / 控风险]
```

### 26.6 实战中的归因报告模板

我一般用Python的`pyfolio`和`alphalens`来生成归因报告。下面是一个简化版的报告模板：

```python
# 收益归因报告模板（简化版）
import pandas as pd
import numpy as np

def generate_attribution_report(portfolio_returns, benchmark_returns,
                                 factor_exposures, factor_returns):
    """
    生成完整的收益归因报告
    """
    report = {}

    # 1. Brinson归因
    # 假设只有两个行业（简化）
    config_return = 0.02  # 配置收益
    select_return = 0.015  # 选择收益
    report['brinson'] = {
        '配置收益': config_return,
        '选择收益': select_return,
        '交互收益': 0.001,
        '超额收益': config_return + select_return + 0.001
    }

    # 2. Barra归因
    factor_contribution = factor_exposures.dot(factor_returns)
    specific_return = portfolio_returns.mean() - factor_contribution.sum()
    report['barra'] = {
        '因子贡献': factor_contribution.to_dict(),
        '特异收益': specific_return
    }

    # 3. 暴露分解
    report['exposure'] = {
        '因子暴露度': factor_exposures.mean().to_dict(),
        '风险贡献占比': (factor_exposures.abs().sum() /
                        factor_exposures.abs().sum().sum()).to_dict()
    }

    return report
```

> **关键提醒：**归因报告不是一次性工作。我建议**每周/每月**跑一次，监控因子暴露的变化。如果发现某个因子的暴露突然变大，一定要去查原因——可能是数据问题，也可能是市场风格切换。

### 26.7 避坑指南：归因分析中的常见错误

做归因这几年，我踩过的坑不少。挑几个典型的说说：

- **忽略交易成本**：归因用的是理论收益，但实盘有滑点和佣金。我曾经有个策略，归因显示超额收益2%，扣掉成本只剩0.3%。
- **因子收益估计偏差**：用OLS回归估计因子收益时，如果因子高度相关，估计值会很不稳定。我建议用**加权最小二乘**或者**正则化回归**。
- **时间窗口选择**：归因周期太短（比如日频），噪声太大；太长（比如年频），又看不到短期变化。我个人习惯用**月频**，兼顾稳定性和时效性。
- **基准选择不当**：基准一定要跟策略风格匹配。你做小盘策略，拿沪深300做基准，归因结果肯定难看。这不是策略的问题，是基准选错了。

> **我的习惯：**每次跑完归因，我都会问自己三个问题：① 超额收益的主要来源是什么？② 这些来源可持续吗？③ 如果市场风格反转，我的组合会怎样？想清楚这三个问题，才算真正理解了你的策略。
