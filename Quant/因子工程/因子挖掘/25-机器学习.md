# 第二十七章 机器学习因子挖掘：从线性回归到神经网络

机器学习做因子挖掘，说白了就是让模型自己去找规律。

传统因子靠人工经验，机器学习因子靠数据驱动。我个人习惯把这两者结合着用——人工因子做底仓，机器学习因子做增强。今天我们就来聊聊五种最常见的机器学习因子：线性回归、决策树、随机森林、XGBoost，还有神经网络。

> **核心观点：** 机器学习因子不是黑箱，而是帮你发现人工难以察觉的非线性关系。但前提是——你得理解它在干什么。

## 5.1 线性回归因子：最朴素的起点

线性回归，说白了就是找一条线，让这条线尽可能拟合数据。

在量化里，我们通常用过去 N 天的特征（比如动量、波动率、成交量）去预测未来一天的收益。公式很简单：

```python
import pandas as pd
from sklearn.linear_model import LinearRegression

# 假设df是日频数据，包含特征列和未来收益列
features = ['momentum_5d', 'volatility_10d', 'volume_ratio']
X = df[features]
y = df['future_return_1d']

model = LinearRegression()
model.fit(X, y)

# 因子值 = 模型预测值
df['lr_factor'] = model.predict(X)
```

我在项目中遇到过一个问题：线性回归对异常值特别敏感。有一次我跑回测，发现因子收益突然飙升，查了半天发现是某只股票因为分红导致价格异常，模型直接学歪了。

> **避坑指南：** 用线性回归做因子，一定要先做去极值和标准化。我曾经因为偷懒没做，结果模型在样本外直接崩了。

## 5.2 决策树因子：非线性关系的入门

决策树能捕捉非线性关系。比如「当动量大于5%且成交量放大时，收益为正」这种规则，决策树天然就能学到。

```python
from sklearn.tree import DecisionTreeRegressor

dt = DecisionTreeRegressor(max_depth=3, min_samples_leaf=100)
dt.fit(X, y)
df['dt_factor'] = dt.predict(X)
```

决策树有个致命问题：容易过拟合。你想想看，如果树太深，它会把噪声也学进去。我建议深度控制在3-5层，叶子节点至少100个样本。

> **注意：** 决策树因子在样本内可能表现很好，但样本外往往回撤很大。原因就是过拟合。我见过有人把深度设到10层，回测曲线漂亮得不行，实盘直接亏到怀疑人生。

## 5.3 随机森林因子：集成学习的威力

随机森林就是很多棵决策树一起投票。每棵树用不同的样本和特征，最后取平均。这样做的好处是——方差降低了，泛化能力更强。

```python
from sklearn.ensemble import RandomForestRegressor

rf = RandomForestRegressor(
    n_estimators=100,
    max_depth=5,
    min_samples_leaf=50,
    random_state=42
)
rf.fit(X, y)
df['rf_factor'] = rf.predict(X)
```

随机森林有个很实用的功能：特征重要性排序。你可以看看哪些特征对预测贡献最大。我记得有一次做因子分析，发现随机森林认为「过去5天收益率」的重要性远高于「过去20天收益率」，这让我重新审视了动量因子的构建逻辑。

> **经验之谈：** 随机森林的树的数量不是越多越好。100棵左右就够了，再多边际收益递减，计算成本却直线上升。

## 5.4 XGBoost因子：梯度提升的王者

XGBoost 是目前最流行的树模型之一。它用梯度提升的方式，一棵树一棵树地修正前一棵树的错误。说白了，就是"错哪里补哪里"。

```python
import xgboost as xgb

model = xgb.XGBRegressor(
    n_estimators=100,
    max_depth=4,
    learning_rate=0.1,
    subsample=0.8,
    colsample_bytree=0.8
)
model.fit(X, y)
df['xgb_factor'] = model.predict(X)
```

XGBoost 有几个关键参数：

| 参数 | 作用 | 建议值 |
| --- | --- | --- |
| learning_rate | 学习率，控制每棵树的贡献 | 0.01-0.1 |
| max_depth | 树深度，控制模型复杂度 | 3-6 |
| subsample | 行采样比例，防过拟合 | 0.7-0.9 |
| colsample_bytree | 列采样比例，增加多样性 | 0.7-0.9 |

我在项目中用 XGBoost 做因子，效果比随机森林好10%-20%。但要注意：XGBoost 调参比较麻烦，建议用网格搜索或者贝叶斯优化来找最优参数。

> **警告：** XGBoost 很容易过拟合，尤其是当特征数量远大于样本数量时。我建议先用特征选择降维，再跑 XGBoost。

## 5.5 神经网络因子：深度学习的尝试

神经网络能拟合极其复杂的非线性关系。在量化里，我们通常用全连接网络（MLP）或者 LSTM 来做因子。

```python
import torch
import torch.nn as nn

class FactorNet(nn.Module):
    def __init__(self, input_dim):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(input_dim, 64),
            nn.ReLU(),
            nn.Dropout(0.2),
            nn.Linear(64, 32),
            nn.ReLU(),
            nn.Dropout(0.2),
            nn.Linear(32, 1)
        )

    def forward(self, x):
        return self.net(x)

```

神经网络因子有个特点：训练时间长，但一旦训练好，预测速度很快。我建议先用小规模数据试跑，确认有效再上全量数据。

> **实用建议：** 神经网络因子不要单独用，最好和树模型因子做集成。我做过实验，神经网络+随机森林的集成因子，夏普比率比单独用任何一个都高。

## 5.7 实战中的选择建议

说了这么多，到底该用哪个？我个人的经验是：

- **起步阶段：** 先用线性回归，快速验证特征是否有效
- **进阶阶段：** 上随机森林或 XGBoost，捕捉非线性关系
- **高阶阶段：** 尝试神经网络，但要做好过拟合控制

记住一点：没有最好的模型，只有最适合你数据的模型。我见过有人用线性回归跑出比神经网络还好的效果，原因就是数据量太小，神经网络根本学不到东西。

> **最后的话：** 机器学习因子不是万能药。它需要和人工因子结合，需要严格的回测验证，更需要你对市场有深刻的理解。工具只是工具，真正决定成败的，还是使用工具的人。
