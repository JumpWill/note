## CSI 主流方案对比

| 维度 | Ceph (Rook) | Longhorn | OpenEBS (Mayastor) |
| --- | --- | --- | --- |
| 类型 | 统一存储（块 / 文件 / 对象） | 分布式块 | 分布式块 |
| 协议 | librbd / kernel krbd / ceph-fuse | iSCSI（默认）/ NVMe-oF | NVMe-oF（Mayastor）/ iSCSI（cStor） |
| 共享（RWX） | ✅ RBD RWO + CephFS RWX | ❌（除非用 NFS） | ❌（除非配 NFS） |
| 架构 | 中心化集群 + CRUSH | 每 volume 一组 engine pod | 每 volume nexus + replica pod |
| 副本 | 3 默认 / EC 可 | 3 默认 | 1 / 2 / 3 默认 |
| 性能 | 中（副本同步 IO） | 中（iSCSI 开销） | **高（io_uring + SPDK）** |
| 部署复杂度 | 高（MON/OSD/MGR/MDS） | **低**（Helm 一键） | 中（Helm 但需要裸盘 / NVMe） |
| 学习曲线 | 陡 | 平 | 中 |
| 运维成本 | 高 | **低** | 中 |
| 适用规模 | **大集群**（>10 节点） | 中（<500 节点） | 中（<200 节点） |
| 快照 / 备份 | ✅ | ✅ + S3 backup | ✅ + S3 / Velero |
| 数据面 | OSD（RADOS） | iSCSI / NVMe-oF | io_uring / SPDK |
| 跨集群复制 | RBD mirror / CephFS mirror | 弱 | 弱 |
| 商业支持 | Red Hat ODF | SUSE / Rancher | DataCore |

## 性能参考

| 场景 | Ceph RBD | Longhorn | OpenEBS Mayastor |
| --- | --- | --- | --- |
| 单盘 4K 随机读 | ~50K IOPS | ~60K IOPS | **~200K IOPS** |
| 单盘 4K 随机写 | ~20K IOPS | ~30K IOPS | **~80K IOPS** |
| 顺序读（NVMe） | ~1 GB/s | ~1 GB/s | **~2 GB/s** |
| 延迟 | 1–5 ms | 1–5 ms | **~100–500 μs** |
| PostgreSQL OLTP | 中 | 中 | **高** |
| 大文件顺序写 | 高 | 高 | 高 |

> 数据基于 NVMe 节点 + 3 副本对比，实际取决于硬件。

## 一句话定位

* **Ceph (Rook)**：通用大集群统一存储，**功能最全，运维最重**
* **Longhorn**：K8s 体验最好、部署最简，**中小集群首选**
* **OpenEBS Mayastor**：性能最强，**数据库上 K8s 首选**

## 决策树

```
要 RWX 共享文件？
  ├─ 是 → CephFS（Rook）或 JuiceFS
  └─ 否（RWO / 块）
       ├─ 集群大（>500 节点）+ 多协议 → Ceph
       ├─ 中小集群 + 简单运维 → Longhorn
       └─ 追求性能（DB on K8s） → OpenEBS Mayastor
```

## 跟云厂商托管存储对比

云上 K8s（EKS / AKS / GKE）一般直接用云盘：

* **AWS**：EBS（块）/ EFS（NFS）/ FSx
* **Azure**：Azure Disk / File
* **GCP**：Persistent Disk / Filestore
* **阿里云**：云盘 / NAS / CPFS

* ✅ 免运维、性能稳定、有 SLA
* ❌ 锁云、跨云 / 混合云不便、自定义策略受限
* ❌ 跨集群 DR 麻烦

**裸金属 / 自建 K8s 才需要上面这些方案**。

## 一致选型建议

* **小团队 + 不想运维**：直接 Longhorn
* **大集群 + 多场景**：Rook + Ceph（RBD + CephFS + RGW）
* **数据库上 K8s**：OpenEBS Mayastor 或本地 NVMe 直挂
* **跨云 / 混合云**：Ceph（RBD mirror / CephFS mirror）或 Longhorn + S3 backup
* **自建对象存储**：Ceph RGW 或 MinIO（注意 MinIO 不是 CSI，是 S3）

## 一句话选型

**通用 → Longhorn；性能 → OpenEBS Mayastor；规模 + 多协议 → Ceph**。
