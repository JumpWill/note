## Cloud Provider 主流方案对比

| 维度 | AWS | 阿里云 | vSphere |
| --- | --- | --- | --- |
| 适用环境 | AWS 公有云 | 阿里云公有云 | VMware 私有云 |
| 官方产品 | EKS | ACK | vSphere Tanzu |
| 托管 K8s 自带 | ✅ EKS | ✅ ACK | ✅ TKGs |
| 自建 K8s 也能用 | ✅ | ✅ | ✅（CPI + CSI） |
| 节点 lifecycle | ✅ ASG | ✅ ESS | ✅ VM 创建/删除 |
| LoadBalancer | ✅ ELB / NLB / ALB | ✅ SLB | ⚠️ 需 NSX-T / 外部 |
| 块存储 CSI | ✅ EBS | ✅ 云盘 | ✅ VMDK / vSAN / vVols |
| 文件存储 | ✅ EFS | ✅ NAS | ✅ NFS |
| 对象存储 | ✅ S3 | ✅ OSS | ❌ |
| 鉴权 | IRSA（IAM Roles for ServiceAccounts） | RAM 角色 | vCenter 账号 |
| 跨云迁移 | ❌ 绑 AWS | ❌ 绑阿里云 | ❌ 绑 VMware |
| 文档 / 中文 | 强 | **最强** | 中 |
| 商业支持 | AWS Support | 阿里云工单 | VMware Support |

## 一句话定位

* **AWS**：AWS 云 / EKS 用户
* **阿里云**：阿里云 / ACK 用户
* **vSphere**：私有云 / VMware 用户

## 决策树

```
云环境？
  ├─ AWS → AWS Cloud Provider
  ├─ 阿里云 → 阿里云 Cloud Provider
  ├─ Azure → azure-cloud-provider（Azure 自己)
  ├─ GCP → gcp-cloud-provider
  ├─ 腾讯云 → 腾讯云 cloud-controller-manager
  ├─ 华为云 → 华为云 CCI / CCE
  ├─ 私有云 VMware → vSphere
  ├─ 私有云 OpenStack → openstack-cloud-provider
  └─ 裸金属 / 自建 → 不用 cloud-provider（或仅 basic node-labels）
```

## 几个细节

### Service type=LoadBalancer

所有公有云都支持，但**配法、annotation、性能都不同**：

* AWS：NLB（4 层）/ ALB（7 层），要选对 annotation
* 阿里云：SLB，annotation 走 `alicloud-loadbalancer-*`
* vSphere：要 NSX-T / MetalLB 之类外部方案

### CSI Driver 跟 cloud-provider 分开

* AWS EBS CSI driver 是独立组件（`aws-ebs-csi-driver`）
* 阿里云 `csi-plugin` 也是独立 DaemonSet
* vSphere CSI 也是独立项目

部署时要**两个都装**（cloud-provider + 各自的 CSI driver）。

### K8s 1.28+ in-tree 已废弃

```yaml
# 老: kubelet 启动参数
--cloud-provider=aws

# 新: 配 provider id + 装 cloud-controller-manager
```

## 私有云常见方案

| 私有云 | Cloud Provider | CSI |
| --- | --- | --- |
| VMware vSphere | cloud-provider-vsphere | vSphere CSI |
| OpenStack | openstack-cloud-provider | cinder-csi-plugin |
| 裸金属 / 自建 | ❌ 不用 | local-path / Rook |
| Nutanix | nutanix-cloud-controller | nutanix-csi |
| H3C / 华为 FusionCompute | 厂商定制 | 厂商定制 |

## 裸金属集群

裸金属 / 自建机房**没有 cloud-provider**：

* Service type=LoadBalancer：用 **MetalLB**（L2 / BGP 模式）
* 持久化：用 **Rook Ceph / Longhorn / OpenEBS**
* 节点 lifecycle：自己写 Operator / 用 IaaS 自动化
* 类似云体验：可以用 **Cluster API**（CAPI）做声明式集群 lifecycle

## 一句话选型

**云上跑 K8s → 用对应云厂商的 cloud-provider；私有云 → vSphere / OpenStack + MetalLB + 分布式存储**。
