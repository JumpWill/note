## Device Plugin 主流方案对比

| 维度 | NVIDIA Device Plugin | SR-IOV Device Plugin | DRA |
| --- | --- | --- | --- |
| 资源类型 | GPU（NVIDIA） | 网卡 VF | 任意（GPU / NIC / FPGA / TPU） |
| API 风格 | 静态 `nvidia.com/gpu: N` | 静态 `intel.com/sriov_net_A: 1` | 动态 ResourceClaim + DeviceClass |
| 共享 | TimeSlicing / MIG | ❌（VF 独占） | ✅ 原生 partial / sharing |
| 拓扑感知 | 弱（Node label + GFD） | 弱 | ✅ scheduler plugin 内置 |
| 调度感知 | ❌ scheduler 不懂 GPU | ❌ | ✅ |
| 参数化 | ❌ | ❌ | ✅ CEL 表达式 |
| 资源声明灵活度 | 低 | 低 | 高 |
| K8s 版本 | 任意 | 任意 | 1.30+ GA, 1.34+ 默认 |
| 生态成熟度 | **最成熟** | 成熟 | **早期**（但官方方向） |
| 硬件依赖 | NVIDIA GPU | 网卡 + 硬件支持 SR-IOV + IOMMU | 厂商出 driver |

## 一句话定位

* **NVIDIA Device Plugin**：CUDA / 训练 / 推理默认
* **SR-IOV Device Plugin**：低延迟网络 / NFV / DPDK
* **DRA**：**新集群首选**，未来统一接口

## 决策树

```
要 GPU？
  ├─ 用 NVIDIA CUDA → NVIDIA Device Plugin
  ├─ 想要复杂 GPU 选型（型号 / 显存 / 拓扑） → DRA
  └─ AMD / Intel / 其他 → 各厂商 Device Plugin 或 DRA driver

要高速网络？
  ├─ NFV / DPDK / 低延迟 → SR-IOV Device Plugin
  └─ 通用 RDMA → DRA + Mellanox DRA driver

新集群（K8s ≥1.30）：
  └─ 一律考虑 DRA（生态允许的话）
```

## 关键场景推荐

| 场景 | 推荐方案 |
| --- | --- |
| AI 训练（大模型，多卡 NVLink） | DRA + NVIDIA DRA driver |
| AI 推理（中等并发） | NVIDIA Device Plugin + TimeSlicing |
| 大数据 / Spark（多 GPU 任务） | DRA（拓扑感知） |
| NFV 运营商虚拟化 | SR-IOV |
| 高频交易 | SR-IOV + DPDK |
| 旧集群升级 | 继续 Device Plugin |
| 新集群 | **DRA** |

## 跟其他接口的关系

* **DRA 替代 Device Plugin** 的资源声明方式，但 GPU DRA driver 跟 NVIDIA Container Toolkit 还是配套的（驱动 + toolkit 不能少）
* **SR-IOV + Multus**：SR-IOV 是 secondary network，要 Multus 配合
* **DRA + Scheduler**：DRA 自带 scheduler plugin，不需额外配置

## 一句话选型

**NVIDIA GPU → NVIDIA Device Plugin（稳）或 DRA（新）；网络直通 → SR-IOV；新集群优先 DRA**。
